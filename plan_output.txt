# Candidate Insight & Comparison Feature -- Implementation Plan

## 1. Architecture Overview and Design Decisions

### Entry Point

The feature is accessed from the existing `ranked_candidates.html` page. Each candidate card will get a new **"Deep Insight"** button alongside the existing View Resume / Shortlist / Contact buttons. Clicking it navigates to a dedicated full-page comparison view at `/recruiter/job/<job_id>/candidate/<resume_id>/insight`.

### Routing Strategy

Both the `ai_engine_bp` and `recruiter_bp` share the URL prefix `/recruiter`. The new routes belong under `ai_engine_bp` because they are AI-driven analysis features, consistent with where the existing scoring results route lives. The chat API endpoints also go under `ai_engine_bp`.

### Service Layer

Two new service files under `D:/Resume-Job-Matching-AI-Powered-LLMs-/services/`:

- `candidate_insight_service.py` -- Pure analysis logic (diff computation, LLM insight generation, interview question generation, career trajectory analysis)
- `chat_service.py` -- LangChain-based conversational interface with Groq

### Data Storage

Two new MongoDB collections for persistence:

- `candidate_insights` -- Caches generated insights per (job_id, resume_id) pair
- `insight_chats` -- Stores chat sessions and message history

### Token Minimization Strategy

The Groq API charges by token and has rate limits. The entire design uses:
1. Structured JSON context instead of raw text dumps
2. Terse system prompts (under 200 tokens each)
3. Pre-computed diff data passed as structured context so the LLM reasons over summaries rather than raw documents
4. Caching of all LLM outputs in MongoDB so re-visits do not re-call the API
5. Chat context window trimming (last 10 messages only)

### Method Details

**`compute_skills_diff(resume, job)`**

This is the most important diff. It compares the JD `skills.mandatory`, `skills.optional`, and `skills.tools` against the candidate's `skills` list.

```python
def compute_skills_diff(self, resume: Resume, job: JobDescription) -> Dict:
    """Compute skill-by-skill diff between candidate and JD."""
    # Normalize all skill names to lowercase for comparison
    candidate_skills = {}
    for s in (resume.skills or []):
        candidate_skills[s.skill_name.lower().strip()] = {
            'skill_name': s.skill_name,
            'proficiency': s.proficiency,
            'category': s.category,
            'years_of_experience': s.years_of_experience
        }

    mandatory = [s.lower().strip() for s in (job.skills.mandatory or [])] if job.skills else []
    optional = [s.lower().strip() for s in (job.skills.optional or [])] if job.skills else []
    tools = [s.lower().strip() for s in (job.skills.tools or [])] if job.skills else []
    all_required = mandatory + optional

    matched = []
    missing_mandatory = []
    missing_optional = []
    tools_matched = []
    tools_missing = []

    # For each required skill, check if candidate has it
    # Use embedding similarity for fuzzy matching (threshold 0.85)
    for req_skill in mandatory:
        found = self._find_skill_match(req_skill, candidate_skills)
        if found:
            matched.append({**candidate_skills[found], 'required_as': 'mandatory'})
        else:
            missing_mandatory.append(req_skill)

    for req_skill in optional:
        found = self._find_skill_match(req_skill, candidate_skills)
        if found:
            matched.append({**candidate_skills[found], 'required_as': 'optional'})
        else:
            missing_optional.append(req_skill)

    for tool in tools:
        found = self._find_skill_match(tool, candidate_skills)
        if found:
            tools_matched.append(tool)
        else:
            tools_missing.append(tool)

    # Extra skills: candidate has but JD does not mention
    matched_names = {m['skill_name'].lower().strip() for m in matched}
    extra = [s.skill_name for s in (resume.skills or [])
             if s.skill_name.lower().strip() not in matched_names
             and s.skill_name.lower().strip() not in [t.lower() for t in tools_matched]]

    total_required = len(mandatory) + len(optional)
    match_rate = len(matched) / total_required if total_required > 0 else 1.0

    return {
        'matched': matched,
        'missing_mandatory': missing_mandatory,
        'missing_optional': missing_optional,
        'extra': extra,
        'tools_matched': tools_matched,
        'tools_missing': tools_missing,
        'match_rate': match_rate
    }

def _find_skill_match(self, required_skill: str, candidate_skills: Dict,
                       threshold: float = 0.85) -> Optional[str]:
    """Find a matching skill using exact match first, then embedding similarity."""
    # Exact match
    if required_skill in candidate_skills:
        return required_skill

    # Substring match (e.g., "python" matches "python3")
    for cand_name in candidate_skills:
        if required_skill in cand_name or cand_name in required_skill:
            return cand_name

    # Embedding similarity fallback
    req_emb = get_embedding(required_skill)
    best_match = None
    best_score = 0.0
    for cand_name in candidate_skills:
        cand_emb = get_embedding(cand_name)
        sim = cosine_similarity(req_emb, cand_emb)
        if sim > best_score:
            best_score = sim
            best_match = cand_name

    if best_score >= threshold:
        return best_match
    return None
```

```python
def compute_education_diff(self, resume: Resume, job: JobDescription) -> Dict:
    required = []
    if job.qualifications and job.qualifications.education:
        for edu_req in job.qualifications.education:
            required.append({
                'degree': edu_req.degree,
                'field': edu_req.field_of_study,
                'level': edu_req.level.value if edu_req.level else None
            })

    actual = []
    for edu in (resume.education or []):
        actual.append({
            'degree': edu.degree,
            'field': edu.field,
            'institution': edu.institution,
            'grade': edu.grade
        })

    # Reuse constraint matcher for the score and explanation
    match_result = self.cm.education_matcher.match_education(
        resume.education or [],
        (job.qualifications.education if job.qualifications else []) or []
    )

    return {
        'required': required,
        'actual': actual,
        'meets_requirement': match_result.normalized_score >= 0.6,
        'score': match_result.score,
        'explanation': match_result.explanation
    }
```

**`compute_experience_diff(resume, job)`**

Reuses `ExperienceMatcher` internals to get the per-role similarity and relevance classification.

```python
def compute_experience_diff(self, resume: Resume, job: JobDescription) -> Dict:
    required_min = 0.0
    required_max = None
    if job.qualifications and job.qualifications.experience_years:
        required_min = job.qualifications.experience_years.min or 0.0
        required_max = job.qualifications.experience_years.max

    # Calculate per-role similarities
    similarities = self.experience_matcher.calculate_job_title_similarity(
        job.title, resume.experience or []
    )

    relevant_roles = []
    irrelevant_roles = []
    for exp_idx, similarity, years in similarities:
        exp = resume.experience[exp_idx]
        role_info = {
            'title': exp.job_title,
            'company': exp.company,
            'years': round(years, 1),
            'similarity': round(similarity, 2),
            'start_date': exp.start_date,
            'end_date': exp.end_date
        }
        if similarity >= 0.5:
            relevant_roles.append(role_info)
        else:
            irrelevant_roles.append(role_info)

    relevant_years = sum(r['years'] for r in relevant_roles)

    match_result = self.cm.experience_matcher.match_experience(
        resume.experience or [], job
    )

    return {
        'required_years_min': required_min,
        'required_years_max': required_max,
        'actual_relevant_years': round(relevant_years, 1),
        'relevant_roles': sorted(relevant_roles, key=lambda x: x['similarity'], reverse=True),
        'irrelevant_roles': irrelevant_roles,
        'meets_requirement': match_result.normalized_score >= 0.6,
        'score': match_result.score,
        'explanation': match_result.explanation
    }
```

```python
def compute_certification_diff(self, resume: Resume, job: JobDescription) -> Dict:
    required = []
    if job.qualifications and job.qualifications.certifications:
        required = [c.lower().strip() for c in job.qualifications.certifications]

    actual = []
    for cert in (resume.certifications or []):
        actual.append({'title': cert.title, 'issuer': cert.issuer})

    actual_names = [c['title'].lower().strip() for c in actual]

    matched = [r for r in required if any(r in a or a in r for a in actual_names)]
    missing = [r for r in required if r not in matched]

    return {
        'required': required,
        'actual': actual,
        'matched': matched,
        'missing': missing
    }
```

```python
def generate_insights(self, diff_data: Dict, scores: Dict,
                      resume_summary: str, job_title: str) -> Dict:
    """Generate LLM-based insights. Designed for minimum token usage."""

    # Build structured context (much cheaper than dumping raw resume text)
    context = self._build_insight_context(diff_data, scores, resume_summary, job_title)

    system_prompt = (
        "You are a senior technical recruiter. Analyze the candidate-vs-job comparison data. "
        "Respond ONLY in valid JSON. Be concise. Each bullet point max 15 words."
    )

    user_prompt = f"""Comparison data:
{context}

Return JSON:
{{
  "strengths": ["..."],
  "weaknesses": ["..."],
  "red_flags": ["..."],
  "cultural_fit_signals": ["..."],
  "career_trajectory": "2-3 sentences",
  "growth_potential": "2-3 sentences",
  "skill_gap_analysis": "2-3 sentences",
  "hiring_recommendation": "1-2 sentences",
  "confidence": "high|medium|low"
}}"""

    try:
        response = self.groq_client.chat.completions.create(
            model=GROQ_MODEL,
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": user_prompt}
            ],
            temperature=0.3,
            max_tokens=800
        )
        content = response.choices[0].message.content
        tokens = response.usage.total_tokens if response.usage else 0

        # Parse JSON from response (reuse existing extract_json_from_response)
        from services.llm_service import extract_json_from_response
        import json
        json_str = extract_json_from_response(content)
        parsed = json.loads(json_str)
        parsed['tokens_used'] = tokens
        return parsed
    except Exception as e:
        logger.error(f"Insight generation failed: {e}")
        return self._fallback_insights(diff_data, scores)
```

**`get_or_create_insight(job_id, resume_id, recruiter_id, scores)`** -- The main orchestrator:

```python
def get_or_create_insight(self, job_id: str, resume_id: str,
                          recruiter_id: str, scores: Dict,
                          resume_data: Dict, job_data: Dict) -> Dict:
    """Get cached insight or compute fresh. Diff is always computed; LLM is lazy."""
    # Check cache
    existing = self.insights_collection.find_one({
        'job_id': ObjectId(job_id),
        'resume_id': ObjectId(resume_id)
    })
    if existing:
        return existing

    # Parse into Pydantic models
    resume = Resume(**resume_data)
    job = JobDescription(**job_data)

    # Compute diffs (no LLM, fast)
    diff_data = self.compute_full_diff(resume, job)

    # Build document (LLM insights NOT generated yet -- lazy loading)
    doc = {
        'job_id': ObjectId(job_id),
        'resume_id': ObjectId(resume_id),
        'recruiter_id': ObjectId(recruiter_id),
        'skills_diff': diff_data['skills'],
        'education_diff': diff_data['education'],
        'experience_diff': diff_data['experience'],
        'certification_diff': diff_data['certifications'],
        'scores': scores,
        'llm_insights': None,        # Generated on demand via separate endpoint
        'interview_questions': None,  # Generated on demand via separate endpoint
        'created_at': datetime.now(timezone.utc),
        'updated_at': datetime.now(timezone.utc)
    }

    from services.type_convertor import prepare_document_for_mongodb
    doc = prepare_document_for_mongodb(doc)
    self.insights_collection.insert_one(doc)
    return doc
```

The LLM insights and interview questions are generated lazily -- only when the user clicks the respective tab or button on the comparison page. This avoids wasting tokens if the recruiter only wants to see the diff view.

## 5. File: `services/chat_service.py`

This file uses LangChain with Groq for the conversational interface.

```python
import os
import logging
from datetime import datetime, timezone
from typing import Dict, List, Optional
from bson import ObjectId

from langchain_groq import ChatGroq
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

logger = logging.getLogger(__name__)

MAX_HISTORY_MESSAGES = 10  # Keep last 10 messages for context window management
MAX_RESPONSE_TOKENS = 400


class CandidateInsightChat:
    """LangChain-based chat for recruiter questions about a candidate vs JD."""

    def __init__(self, db):
        self.db = db
        self.chats_collection = db['insight_chats']
        self._llm = None

    @property
    def llm(self):
        if self._llm is None:
            self._llm = ChatGroq(
                groq_api_key=os.getenv('GROQ_API_KEY'),
                model_name="llama-3.3-70b-versatile",
                temperature=0.3,
                max_tokens=MAX_RESPONSE_TOKENS
            )
        return self._llm

    def _build_system_message(self, insight_data: Dict) -> str:
        """Build a compact system message from the insight data."""
        skills = insight_data.get('skills_diff', {})
        exp = insight_data.get('experience_diff', {})
        edu = insight_data.get('education_diff', {})
        scores = insight_data.get('scores', {})

        matched_skills = [s.get('skill_name', s) if isinstance(s, dict) else s
                         for s in skills.get('matched', [])]
        missing_skills = skills.get('missing_mandatory', []) + skills.get('missing_optional', [])

        relevant_roles = [f"{r['title']} at {r['company']} ({r['years']}yr)"
                         for r in exp.get('relevant_roles', [])[:3]]

        context = (
            f"You are a recruitment analyst. Answer questions about this candidate.\n"
            f"Overall match: {scores.get('overall', 0)*100:.0f}%\n"
            f"Skills matched: {', '.join(matched_skills[:10])}\n"
            f"Skills missing: {', '.join(missing_skills[:10])}\n"
            f"Relevant experience: {exp.get('actual_relevant_years', 0)} years. "
            f"Roles: {'; '.join(relevant_roles)}\n"
            f"Education: {edu.get('explanation', 'N/A')}\n"
            f"Be concise. Max 3 sentences per answer unless asked for detail."
        )
        return context

    def _get_or_create_session(self, job_id: str, resume_id: str,
                                recruiter_id: str) -> Dict:
        """Get existing chat session or create new one."""
        session = self.chats_collection.find_one({
            'job_id': ObjectId(job_id),
            'resume_id': ObjectId(resume_id),
            'recruiter_id': ObjectId(recruiter_id)
        })
        if not session:
            session = {
                'job_id': ObjectId(job_id),
                'resume_id': ObjectId(resume_id),
                'recruiter_id': ObjectId(recruiter_id),
                'messages': [],
                'total_tokens_used': 0,
                'created_at': datetime.now(timezone.utc),
                'updated_at': datetime.now(timezone.utc)
            }
            self.chats_collection.insert_one(session)
        return session

    def send_message(self, job_id: str, resume_id: str, recruiter_id: str,
                     user_message: str, insight_data: Dict) -> Dict:
        """Process a user message and return the AI response."""
        session = self._get_or_create_session(job_id, resume_id, recruiter_id)

        # Build message history (trimmed to last N messages)
        history = session.get('messages', [])[-MAX_HISTORY_MESSAGES:]

        # Construct LangChain messages
        system_msg = SystemMessage(content=self._build_system_message(insight_data))
        messages = [system_msg]

        for msg in history:
            if msg['role'] == 'user':
                messages.append(HumanMessage(content=msg['content']))
            else:
                messages.append(AIMessage(content=msg['content']))

        messages.append(HumanMessage(content=user_message))

        try:
            response = self.llm.invoke(messages)
            ai_content = response.content
            tokens_used = 0
            if hasattr(response, 'response_metadata'):
                usage = response.response_metadata.get('token_usage', {})
                tokens_used = usage.get('total_tokens', 0)
        except Exception as e:
            logger.error(f"Chat LLM error: {e}")
            ai_content = "I'm sorry, I encountered an error processing your question. Please try again."
            tokens_used = 0

        # Store both messages
        now = datetime.now(timezone.utc)
        new_messages = [
            {'role': 'user', 'content': user_message, 'timestamp': now, 'tokens_used': 0},
            {'role': 'assistant', 'content': ai_content, 'timestamp': now, 'tokens_used': tokens_used}
        ]

        self.chats_collection.update_one(
            {'_id': session['_id']},
            {
                '$push': {'messages': {'$each': new_messages}},
                '$inc': {'total_tokens_used': tokens_used},
                '$set': {'updated_at': now}
            }
        )

        return {
            'response': ai_content,
            'tokens_used': tokens_used,
            'total_messages': len(history) + 2
        }

    def get_chat_history(self, job_id: str, resume_id: str,
                         recruiter_id: str) -> List[Dict]:
        """Get full chat history for display."""
        session = self.chats_collection.find_one({
            'job_id': ObjectId(job_id),
            'resume_id': ObjectId(resume_id),
            'recruiter_id': ObjectId(recruiter_id)
        })
        if not session:
            return []
        return session.get('messages', [])

    def clear_chat(self, job_id: str, resume_id: str, recruiter_id: str) -> bool:
        """Clear chat history for a session."""
        result = self.chats_collection.update_one(
            {
                'job_id': ObjectId(job_id),
                'resume_id': ObjectId(resume_id),
                'recruiter_id': ObjectId(recruiter_id)
            },
            {'$set': {'messages': [], 'total_tokens_used': 0,
                      'updated_at': datetime.now(timezone.utc)}}
        )
        return result.modified_count > 0
```

```html
                        onComplete(data.result);

                appendChatMessage('assistant', data.response);
```

## 11. Key Design Trade-offs and Rationale

**Why lazy-load LLM content instead of generating on page visit?**
The diff view is the most useful view and costs zero tokens. Many recruiters will use only the diff view. Generating LLM insights preemptively would waste tokens and slow down the initial page load. The lazy approach means a recruiter who visits 50 candidate comparison pages but only generates insights for 5 pays for only 5 LLM calls.

**Why use raw Groq client for insights instead of LangChain?**
LangChain adds overhead (both latency and token usage from prompt templates). The insight generation and question generation are single-shot calls with no conversation state. Raw Groq client calls are simpler, faster, and use fewer tokens. LangChain is used only for the chat interface where its message history management adds genuine value.

**Why store diffs in MongoDB instead of computing on every page load?**
The diff computation calls the embedding service (via `_find_skill_match`), which itself makes HuggingFace API calls (cached in MongoDB, but still). Storing the computed diff means subsequent page visits are instant with no external API calls.

**Why not use FAISS for skill matching?**
The project already removed FAISS (per git history). The embedding cache in MongoDB combined with the small number of skills per resume (typically 10-30) makes a linear scan with cached embeddings fast enough. FAISS would be over-engineering for this scale.

**Why a task polling approach instead of SSE/WebSockets for LLM generation?**
The existing project uses the `TaskRunner` with polling (see how `run_ai_engine` works). Adding SSE or WebSockets would introduce new infrastructure concerns. Polling every 2 seconds for a task that takes 3-5 seconds is simple and consistent with the existing pattern.
